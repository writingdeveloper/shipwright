# @repo/i18n — Claude Code rules

next-intl routing + locale-aware navigation FACTORIES. The MECHANISM lives here; the STRINGS + locale POLICY live in the app (`apps/web/messages/<locale>.json`, `apps/web/i18n/routing.ts`).

- **Factories, not singletons** (`src/routing.ts`, `src/navigation.ts`): this package exports `createRouting` + `createI18nNavigation` (thin re-exports of next-intl's `defineRouting` / `createNavigation`). The APP owns its locale policy — `apps/web/i18n/routing.ts` calls `createRouting({ locales, defaultLocale, localePrefix: "as-needed" })` and is the single source of truth for that app's locales. `as-needed` keeps the default locale unprefixed (`/`, `/sign-in`), others prefixed (`/ko/...`). Add a locale = edit the APP's `i18n/routing.ts` AND add `apps/web/messages/<locale>.json`; the app drift-guard (`apps/web/tests/messages.test.ts`) fails until their key sets match.
- **Navigation**: the app builds its instance in `apps/web/i18n/navigation.ts` (`createI18nNavigation(routing)` from the **`@repo/i18n/navigation`** subpath) and app code imports the locale-aware `Link` / `redirect` / `usePathname` / `useRouter` / `getPathname` from THAT app-local module (relative path) — NOT `next/link` / `next/navigation` (they drop the `/ko` prefix), and NOT `@repo/i18n` (it no longer exports bound helpers, so a leftover import is a compile error).
- **`tsconfig` `declaration: false`** is intentional: next-intl's `createNavigation` infers types through `.pnpm` paths that can't be named in a `.d.ts` (TS2742). The app consumes `src` directly via `transpilePackages`, so no declarations are emitted.
- **Build scripts**: next-intl pulls `@swc/core` + `@parcel/watcher` (native build scripts). pnpm 11.3 reads build allow/deny from `pnpm-workspace.yaml` `allowBuilds` (NOT the package.json `pnpm` field, which it ignores) — both are set `false` (unused at runtime; like esbuild/sharp).
- **proxy composition**: locale routing is isolated behind the app's `apps/web/i18n/proxy.ts` `applyI18n()` seam, composed with the nonce CSP + rate-limit in `apps/web/proxy.ts`. The matcher MUST exclude `.*\..*` so file routes (e.g. `/sw.js`) aren't locale-rewritten (which broke service-worker registration once). To opt out of i18n, `applyI18n` becomes `NextResponse.next()` and `withCsp` handles it unchanged.
- **Messages + locale policy are app-owned** (`apps/web/messages/`, `apps/web/i18n/`); this package owns only the factory mechanism and no longer reaches into the app. The drift-guard lives with what it guards: `apps/web/tests/messages.test.ts`.
