# @repo/payments — Claude Code rules

Stripe subscription billing. Owns the Stripe client, hosted-Checkout creation, webhook verification + idempotent processing, and owner-scoped subscription reads.

- **Server-only** entry (`@repo/payments`): reads server secrets (`STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRICE_ID`) and the libSQL `db`. Import it from a Server Action / Route Handler, never a client component. Use `@repo/payments/config` (client-safe: `publicPriceId`, `hasPublicPriceId`) for client rendering decisions.
- **Graceful degrade is the core invariant**: every Stripe var is OPTIONAL via `@repo/env`. With no `STRIPE_SECRET_KEY` the Stripe client is NEVER constructed — `createCheckoutSession` returns `{ configured: false }` and `constructWebhookEvent` throws a clean "not configured" the route maps to 503. The app/tests/CI run with zero keys. Use `isBillingConfigured()` (secret key AND a price id) to gate UI.
- **Hosted Checkout only** (redirect flow), never embedded Stripe.js — so there is no script to whitelist and the app's nonce CSP is untouched. `createCheckoutSession` puts `userId` in `client_reference_id` + `metadata` for webhook attribution.
- **Webhook (`handleWebhookEvent`) is idempotent**: Stripe retries the same `event.id` on any non-2xx, so dedupe by `event.id` via the `processed_stripe_event` table (skip if seen), record the id LAST (so a crash mid-processing re-runs on retry). Handles `checkout.session.completed` + `customer.subscription.updated/deleted`; ignores+records the rest. `constructWebhookEvent` verifies the signature with `constructEventAsync` and throws on a bad signature.
- **Schema** lives in `@repo/db` (`processed_stripe_event`, `subscription`). Subscription reads are owner-scoped by `userId` (the DB cascade is not an authz boundary). Apply with `pnpm --filter @repo/db db:push` (absolute `DATABASE_URL`).
- `apiVersion` is intentionally omitted from the Stripe client so the SDK uses its pinned default; pin via the `stripe` dependency version instead.
- The web app route is `apps/web/app/api/stripe/webhook/route.ts` (raw body, ACK fast, process in `after()`); the dashboard billing UI is `app/dashboard/{billing-actions.ts,upgrade-button.tsx}` + the billing card in `page.tsx`.
