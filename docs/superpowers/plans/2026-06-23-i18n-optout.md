# i18n opt-out + app-level locale config тАФ Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.
> Spec: `docs/superpowers/specs/2026-06-23-i18n-optout-design.md`.

**Goal:** Invert locale policy from `@repo/i18n` (singletons) to the app
(`apps/web/i18n/*` instances), via `createRouting`/`createI18nNavigation`
factories, and add a proxy removal seam тАФ zero behavior change.

**Architecture:** `@repo/i18n` тЖТ pure factory lib (no singletons, so a missed
import is a compile error). App owns instances + locale list + drift-guard.

**Tech:** next-intl 4, Next 16 App Router, relative imports (no `@/*` alias).

**Nature:** one **atomic refactor** тАФ removing the singletons breaks every
consumer until rewritten, so there is no meaningful green intermediate state.
Verified by the full gate (check-types is the safety net) + e2e, then one commit.

## Task 1: `@repo/i18n` тЖТ factory library

**Files:** `packages/i18n/src/{routing,navigation,index}.ts`, `test/routing.test.ts`, `CLAUDE.md`

- `src/routing.ts`:
```ts
import { defineRouting } from "next-intl/routing";

type RoutingInput = {
  locales: readonly string[];
  defaultLocale: string;
  localePrefix?: Parameters<typeof defineRouting>[0]["localePrefix"];
};

/**
 * Factory for a next-intl routing config. The app owns the locale POLICY and
 * calls this from `apps/web/i18n/routing.ts`; this package owns only the
 * MECHANISM. Defaults `localePrefix` to "as-needed" (default locale unprefixed,
 * others prefixed) тАФ the starter convention тАФ but the app may override it.
 */
export function createRouting(config: RoutingInput) {
  return defineRouting({ localePrefix: "as-needed", ...config });
}

export type Routing = ReturnType<typeof createRouting>;
```
- `src/navigation.ts`:
```ts
import { createNavigation } from "next-intl/navigation";

import type { Routing } from "./routing";

/**
 * Factory for locale-aware navigation APIs (Link / redirect / usePathname /
 * useRouter / getPathname) bound to the app's routing instance. Call from
 * `apps/web/i18n/navigation.ts`. Use THESE, not `next/*`, for internal links so
 * the active locale prefix is preserved.
 */
export function createI18nNavigation(routing: Routing) {
  return createNavigation(routing);
}
```
- `src/index.ts`:
```ts
/**
 * @repo/i18n тАФ next-intl routing + locale-aware navigation FACTORIES.
 *
 * Owns the MECHANISM only. The app owns POLICY (its locale list) and creates the
 * instances: `apps/web/i18n/routing.ts` (createRouting) +
 * `apps/web/i18n/navigation.ts` (createI18nNavigation). Removing the old
 * `routing`/`locales`/`defaultLocale` singletons is deliberate: every consumer
 * now resolves through app-local modules, so a missed migration is a COMPILE
 * error, not a silently dropped locale prefix.
 */
export { createRouting, type Routing } from "./routing";
```
- `test/routing.test.ts` тЖТ factory test:
```ts
import { describe, expect, it } from "vitest";

import { createRouting } from "../src/routing";

describe("createRouting", () => {
  it("applies the as-needed default and passes locales through", () => {
    const routing = createRouting({ locales: ["en", "ko"], defaultLocale: "en" });
    expect(routing.locales).toEqual(["en", "ko"]);
    expect(routing.defaultLocale).toBe("en");
    expect(routing.localePrefix).toBe("as-needed");
  });

  it("lets the app override localePrefix", () => {
    const routing = createRouting({
      locales: ["en"],
      defaultLocale: "en",
      localePrefix: "always",
    });
    expect(routing.localePrefix).toBe("always");
  });
});
```

## Task 2: app i18n instances (`apps/web/i18n/`)

- `routing.ts` (NEW):
```ts
import { createRouting } from "@repo/i18n";

/** This app's locale POLICY тАФ the single source of truth. Add a locale here AND
 *  add `messages/<locale>.json`; tests/messages.test.ts fails until they match. */

export type Locale = (typeof routing.locales)[number];
export const { locales, defaultLocale } = routing;
```
- `navigation.ts` (NEW):
```ts
import { createI18nNavigation } from "@repo/i18n/navigation";

import { routing } from "./routing";

export const { Link, redirect, usePathname, useRouter, getPathname } =
  createI18nNavigation(routing);
```
- `proxy.ts` (NEW) тАФ removal seam:
```ts
import { type NextRequest, NextResponse } from "next/server";
import createMiddleware from "next-intl/middleware";

import { routing } from "./routing";

const intlMiddleware = createMiddleware(routing);

/**
 * i18n locale-routing seam for the proxy. To remove i18n, replace the body with
 * `return NextResponse.next();` and drop the next-intl import тАФ `withCsp` handles
 * a plain next() unchanged.
 */
export function applyI18n(request: NextRequest): NextResponse {
  return intlMiddleware(request);
}
```
- `request.ts`: `import { routing } from "@repo/i18n"` тЖТ `"./routing"`.
